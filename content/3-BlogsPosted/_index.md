###  [Blog 1 - AMAZON RDS LOG ANALYSIS USING NATURAL LANGUAGE WITH KIRO AND MCP](3.1-Blog1/)
This blog shares practical perspectives and lessons on applying AI to solve the "nightmare" of manual log reading on Amazon RDS. By combining Amazon CloudWatch with the MCP protocol and the Kiro AI assistant, system engineers can easily query and analyze root causes from thousands of complex log lines using only natural language commands. This is an effective solution that helps reduce the load on the operations team, accelerates incident response time, and opens a new direction in cloud database security management.

###  [Blog 2 - DECODING AWS SITE-TO-SITE VPN LOGS FOR NETWORK OBSERVABILITY](3.2-Blog2/)
This blog explores how AWS combines Generative AI and ChatOps to enhance the observability and management of AWS Site-to-Site VPN connections. By leveraging Amazon Bedrock and AWS DevOps Agent, the solution can automatically interpret BGP and IKE logs, identify the root causes of network issues, and provide remediation guidance through email or collaboration platforms such as Slack. This approach simplifies network operations, improves troubleshooting efficiency, and significantly reduces incident resolution time.

###  [Blog 3 - PRIORITIZING AWS HEALTH ALERTS USING AWS USER NOTIFICATIONS](3.3-Blog3/)
This blog post delves into how to definitively resolve "notification fatigue" when monitoring operational and maintenance events from AWS Health. By combining the AWS User Notifications service with the automation capabilities of AWS CloudFormation, the system not only automatically filters out noise from unused services but also categorizes the information flow by urgency (sending immediate alerts for critical issues and batched summaries for informational updates). Thanks to this approach, we can maintain high focus on core risks, keep the inbox clean, and easily standardize the management process for both a single account and an entire AWS Organizations setup.
